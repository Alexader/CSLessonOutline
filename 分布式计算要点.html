  1. 分布式计算：分布式计算指在分布式系统上执行的计算。分布式计算是将一个大型计算任务分成很多部分分别交给其他的计算机处理，并将所有的计算结果合并为原问题的解决方案。这里与并行计算不同的是，并行计算是使用多个处理器并行执行单个计算。

    优点：

    • 超大规模 * 虚拟化

    • 高可伸缩性 * 按需服务

  4. CAP理论

  5. 分布式计算的核心知识点：IPC（进程间通信）

    • 概念：互相独立的进程（进程是程序的运行时表示）间通信及共同协作完成某项任务的能力。

    • 当一个进程与另一个进程进行通信时，IPC被称为单播（unicast）；当一个进程与另外一组进程进行通信时，IPC被称为组播（multicast）。

    • 类型：

    • 接口原型：send(),receive(),connect(),disconnect();

    • IPC 的事件同步和异步：通信一方需要等待另外完成某一动作之后，收到通知，才能开始执行，这之前处于阻塞的状态，这是事件同步。如果一方进程不需要等待另一方完成某一任务，只需要在收到数据到达的通知时去处理即可，这是异步通信。

    • 死锁：同步或者说是阻塞按照错误的顺序发起的话，可能会引起死锁。解决的办法

      • 开通子线程。在子线程中运行可能会发生阻塞的任务。
      • 设置超时。

  • 网络服务

    • 工作原理：服务提供者将自身注册到网络上的目录服务器；当服务请求者（进程）需访问服务时，则在运行时与目录服务器联系；然后，如果请求的服务可用，则目录服务器将向目录服务进程提供一个有关该服务的引用；最后，进程利用该引用来与所需的服务进行交互。
    • 和RMI的区别：服务对象在全局目录服务中注册，允许联邦网络中的服务请求者查询和访问这些服务。网络服务范型提供了一种额外的透明性：位置透明性。请求者无需知道访问对象活服务的位置。

第三章

  • Socket的基本原理

    分成两种socket，分别是流式的和数据报的socket。

  • 两种socket编程的基本步骤

    1. 数据报socket server新建一个DatagramSocket实例指定要监听的端口号，接受数据的时候，新建一个Datagram，调用DatagramSocket.receive()方法。

      client要发送数据的时候，直接创建一个DatagramSocket实例。然后通过新建一个Datagram实例，要指定server的hostname或者IP和端口号，调用DatagramSocket.send()方法发送数据报即可。

      UDP的发送方式很随意，只要把数据装到packat里面，发送出去就不用管了，就很方便。

    2. 流式socket编程

      server端新建一个serversocket()，监听一个端口的信号，client新建一个socket，指定发送的hostname和端口，从这个socket去的输入或者输出流，在采用java中的各种适合的IO流类进行数据传输。

      最重要的就是关注什么时候才会断开读取，很多时候socket虽然连着，但是并没有发送东西，接收方这时候还在一直等待，就会造成阻塞。

  • 会话是什么

    相互关联依赖的一组通信过程。

  • 三层软件开发的方法

    • 应用逻辑
    • 表现逻辑
    • 服务逻辑

  • 面向连接和无连接的服务器

    无连接服务器：使用无连接的IPC API（如数据包socket)；并发的多个会话可以交叉执行。

    面向连接服务器：使用面向连接的进程间通信API（如流式socket)；并发的多个会话一般是被顺序执行的，除非服务器是多线程的。

  • 并发和迭代服务器

    并发服务器和迭代服务器都是面向连接的服务器

    迭代服务器：在面向连接服务器中不存在重叠的客户会话，这是因为服务器在同一时刻，仅能与一位已经接受的客户连接交换数据，这种服务器被称为迭代服务器。即非多线程的连接服务器。

    并服务器：能够并行处理多个客户会话。可以使用线程或异步IPC操作来提供并发服务器。

  • 迭代服务器一定是面向连接的吗？

    不一定，有TCP和UDP迭代服务器，即使不是面向连接的，也可以通过应用层的软件控制为迭代的。

  • 状态服务器

    一般都是无状态的服务器，不会出现性能瓶颈，不需要额外的开销。http是没有状态的。

  • 注册远程调用的服务，服务器实现服务程序，客户端继承远程接口，调用接口

    服务器远程回调，需要客户端注册一个远程程序

  • 桩下载，client不用实现具体的调用函数，而是在stub中下载对应的程序，这样方便维护。

  • 安全管理器(System.setSecurityManager(new RMISecurityManager()))；防止从远程服务器上下载的code执行敏感操作，比如操作文件系统和网络下载。

  • 客户端回调：客户在注册服务器上注册自己的，以便服务器在相应事件发生的时候远程调用客户端方法。

  • stub下载解决的问题

    使用动态stub下载，客户主机上就不再需要有stub类文件的拷贝了。取而代之的是，stub类可以在客户被激活时，按需从Web服务器传送到客户主机上。

  • socket API 和RMI比较

    1. socket api的执行与操作系统密切相关，因此执行开销更少。RMI需要额外的软件支持，包括代理和目录服务，这些不可避免地带来运行时开销。
    2. RMI api提供了使软件开发任务更为简单的抽象。用高级抽象开发的程序更易理解，因此更易调试
    3. 由于运行在底层，SOCKETAPI通常是平台和语言独立的,RMI则不一定。

第六章（P2P）

  • 是什么P2P?

    是一种以非集中化方式使用分布式资源来完成一些关键任务的系统和应用。

  • 网络分类

    1. 第一代P2P网络：中心化拓扑。

      所有的索引都在中心服务器，资源在各个节点中。还是会有中心服务器宕机不可用的问题

    2. 第二代P2P网络：全分布式非结构化->全分布式结构化

      ​

    3. 第一代P2P网络：半分布式拓扑。

      保留了性能较高的节点作为超级节点，用于运行检索算法。

    4. 第一代P2P网络：正在研究的分布式拓扑

      拓扑可扩展性可靠性可维护性搜索算法效率复杂查询
      中心化拓扑差差最好最高支持
      全分布式非结构化拓扑差好最好中支持
      全分布式结构化拓扑好好好高不支持
      半分布式拓扑中中中中支持

  4. 对象存储与传统存储有何不同

    在传统的存储系统中用文件或块作为基本的存储单位，块设备要记录每个存储数据块在设备上的位置；

    而在对象存储系统中，对象是数据存储的基本单元，Object维护自己的属性，从而简化了存储系统的管理任务，增加了灵活性，在存储设备中，所有对象都有一个对象标识，通过对象标识OSD命令访问该对象。
